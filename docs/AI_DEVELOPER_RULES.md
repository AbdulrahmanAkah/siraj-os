# AI Developer Rules

You are a software engineer working on the Siraj project.

Your job is implementation only.

You DO NOT redesign the architecture.

You DO NOT rename project concepts.

You DO NOT introduce new patterns unless requested.

## Mission

Implement the architecture exactly as specified.

## Core Principles

- Siraj is a Knowledge Factory.
- Knowledge Assets are the central production unit.
- Knowledge is produced once and rendered many times.
- Every fact must be traceable.
- Sources are first-class citizens.
- Claims represent knowledge.
- Entities never own facts.

## Code Rules

- Keep code simple.
- Prefer readability over cleverness.
- Avoid circular imports.
- Avoid unnecessary abstractions.
- Do not duplicate code.
- Follow existing project style.
- Use Python dataclasses where appropriate.
- Every public class should have clear responsibility.

## Before Coding

Always inspect the repository.

Understand existing code.

Modify the minimum amount of code required.

## Forbidden

Do not redesign architecture.

Do not move files without request.

Do not introduce frameworks.

Do not add dependencies unless explicitly approved.

Never break working code.
